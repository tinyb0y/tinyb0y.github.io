Previse HTB

tinyb0y on Jan 82022-01-08T05:30:00+05:30
5 min read

An easy box with an unvalidated redirection vulnerability, access to registration page and dump the file sitebackup.zip and exploit a post parameter to get remote code execution on the machine. Use the mysql configuration file and enumerate accounts in database and crack the password to get user on the machine. For root the sudo user can run a script where the attacker can inject the path variable and exploit the `date` function.
  • Nmap
  • Recon
  • Unvalidated Redirect Vulnerability
  • User escalation
  • Root Privilege Escalation

Root Escalation

There are two possiblities, I see how to exploit. One is through changing the PATH variable and point date function to our exploit file and other is to make a softlink on to the files mentioned in the script.

Let’s go with date function which is fairly easy way to do it.

I wrote whoami inside a date file and added it to our PATH variable

File name was attached with root which is our whoami command as it as root user.
